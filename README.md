# Slay the Spire 2 Mod Manager

这是一个用于管理《Slay the Spire 2》模组的绿色版工具，支持自动检测游戏目录、备份/还原 `mods`、从同步源下载并更新模组，以及维护本地固定保留的公共模组。

## 功能

- 自动检测 Steam 游戏目录，找不到时也可以手动选择。
- 一键备份当前 `mods` 和游戏目录下的 zip 文件。
- 从同步源下载 `mods.zip` 并自动解压替换当前 `mods`。
- 同步前要求已存在备份，避免误覆盖本地模组。
- 检测到已有备份和 `mods` 文件夹时，同步前会再次确认覆盖。
- 支持 `mods_common` 公共模组目录：每次同步后会自动复制并覆盖到新的 `mods` 中。
- 支持在设置页打开、添加、刷新、删除公共模组。
- 绿色版 exe，双击即可使用，无需安装。

## 使用方法

1. 打开 `SlayTheSpire2ModManager.exe`。
2. 如果软件没有自动检测到游戏目录，进入“设置”页手动选择《Slay the Spire 2》的游戏目录。
3. 首次同步前，先在“模组管理”页点击“备份”。
4. 在“设置”页维护同步源，可以填写 GitHub 仓库地址或 zip 下载直链。
5. 回到“模组管理”页，点击对应同步源按钮开始同步。
6. 如果提示当前模组会被覆盖，确认后才会继续同步。

## 公共模组 mods_common

如果有一些本地模组希望每次同步后都保留，可以使用公共模组功能。

### 方法一：在软件内添加

1. 进入“设置”页。
2. 点击“公共模组”区域的“添加当前模组”。
3. 选择当前 `mods` 目录下的一级模组文件夹。
4. 软件会复制一份到游戏目录下的 `mods_common`。

### 方法二：手动维护

也可以手动在游戏目录创建或维护 `mods_common` 文件夹。同步完成后，软件会自动把 `mods_common` 里的所有内容复制到新的 `mods` 目录中，同名内容会覆盖同步源里的内容。

## 注意事项

- 同步会替换当前 `mods` 文件夹，请先备份。
- 备份目录格式为 `mods_backup_YYYYMMDDHHMM`。
- 删除备份和公共模组前会弹出确认。
- 同步源下载的 zip 会先校验为有效压缩包，再替换本地 `mods.zip`。

## Release 产物

推送到 `main` 后，GitHub Actions 会自动创建 Release，并上传：

- `mods.zip`：当前仓库中的模组包。
- `SlayTheSpire2ModManager.exe`：可直接运行的绿色版管理器。
